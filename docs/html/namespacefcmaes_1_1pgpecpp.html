Optimize a given objective function by minimizing its value using the PGPE (Policy Gradient with Parameter-based Exploration) method.

This optimization approach uses parallel processing and supports both optional parameter normalization and adaptive learning rates for center and standard deviation.

   fun (Callable[[ArrayLike], float]): Objective function to minimize. Must
       take a single input of type ArrayLike and return a float.
Parameters
boundsBounds for the variables. Should be specified as a tuple (lower_bounds, upper_bounds) or None for unbounded variables.
x0Initial guess for the variables. If None, random initialization will be applied within bounds if specified. input_sigma (Optional[Union[float, ArrayLike, Callable]]): Initial standard deviation for parameter sampling. Defaults to 0.1. Can be a scalar, array, or callable returning an array.
popsizePopulation size for sampling. Defaults to 32. If not specified or odd, it will be adjusted to the next even number.
max_evaluationsMaximum number of function evaluations to perform. Defaults to 100000.
workersNumber of parallel workers to use for evaluation. Defaults to None (no parallelism).
stop_fitnessValue of fitness to stop early. If the objective reaches this value, the algorithm will terminate early. Defaults to -infinity.
rgRandom number generator to use during optimization. Defaults to Generator(PCG64DXSM()).
runidUnique identifier for the run. Useful for distinguishing runs in logging or debugging.
normalizeWhether to normalize the input parameters for the optimizer. Defaults to True.
lr_decay_stepsNumber of steps for learning rate decay. Defaults to 1000.
use_rankingWhether to use ranking over raw fitness when calculating updates. Defaults to True.
center_learning_rateLearning rate used to update the center. Defaults to 0.15.
stdev_learning_rateLearning rate used to update the standard deviation. Defaults to 0.1.
stdev_max_changeMaximum allowed change for standard deviation updates. Defaults to 0.2.
b1Exponential moving average factor for the first moment estimate during adaptive updates. Defaults to 0.9.
b2Exponential moving average factor for the second moment estimate during adaptive updates. Defaults to 0.999.
epsSmall term added to avoid division by zero in adaptive algorithms. Defaults to 1e-8.
decay_coefCoefficient that controls decay in the learning rate updates. Defaults to 1.0.
Returns
OptimizeResult An object containing optimization results, including the best parameters found (x), the objective value (fun) at those parameters, the number of function evaluations (nfev), the number of iterations (nit), the stopping status as an integer code (status), and a boolean success flag (success).
