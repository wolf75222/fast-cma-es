15 Description:
16 - Numpy based implementation of CVT MAP-Elites including CMA-ES emitter and CMA-ES drilldown.
17
18 See [2] and [3]
19
20 MAP-Elites implementations differ in the following details:
21
22 1) Initialisation of the behavior space:
23
24 a) Generated from some solution distribution by applying the fitness function to determine their behavior.
25 b) Generated from uniform samples of the behavior space.
26
27 We use b) because random solutions may cover only parts of the behavior space. Some parts may only be reachable
28 by optimization. Another reason: Fitness computations may be expensive. Therefore we don't compute fitness
29 values for the initial solution population.
30
31 2) Initialization of the niches:
32
33 a) Generated from some solution distribution.
34 b) Generated from uniform samples of the solution space. These solutions are never evaluated but serve as
35 initial population for SBX or Iso+LineDD. Their associated fitness value is set to math.inf (infinity).
36
37 We use b) because this way we:
38 - Avoid computing fitness values for the initial population.
39 - Enhance the diversity of initial solutions emitted by SBX or Iso+LineDD.
40
41 3) Iso+LineDD [4] is implemented but doesn't work well with extremely ragged solution
42 landscapes. Therefore SBX+mutation is the default setting.
43
44 4) SBX (Simulated binary crossover) is taken from mode.py and simplified. It is combined with mutation.
45 Both spread factors - for crossover and mutation - are randomized for each application.
46
47 5) Candidates for CMA-ES are sampled with a bias to better niches. As for SBX only a subset of the archive is
48 used, the worst niches are ignored.
49
50 6) There is a CMA-ES drill down for specific niches - in this mode all solutions outside the niche
51 are rejected. Restricted solution box bounds are used derived from statistics maintained by the archive
52 during the addition of new solution candidates.
53
54 7) The QD-archive uses shared memory to reduce inter-process communication overhead.

114 """
115 Optimizes a map-elites evolutionary algorithm to find diverse and high-performing solutions
116 by distributing samples into niches, iteratively evolving them, and maintaining an archive
117 to store the found solutions.
118
119 This function leverages a map-elites approach to optimize solutions across a given feature
120 space or behavioral space defined by the `qd_bounds` parameter. The objective of the optimization
121 is to populate an archive with solutions that exhibit high fitness while ensuring diversity
122 in behavior across the space. The archive is continuously updated and sorted during the iterations
123 to reflect the best solutions in each niche. The performance is monitored throughout the
124 optimization process.
125
126 Args:
127 qd_fitness (Callable[[ArrayLike], Tuple[float, np.ndarray]]): The quantitative fitness
128 function used to evaluate the solutions. It must return a tuple containing the fitness
129 score and the corresponding feature descriptor/behavior representation.
130 bounds (Bounds): Represents the variable bounds for the optimization space.
131 qd_bounds (Bounds): Defines the range of the quality-diversity feature/behavioral space.
132 niche_num (Optional[int]): The total number of niches in the archive. Determines how
133 the solutions will be distributed across the feature space. Defaults to 4000.
134 samples_per_niche (Optional[int]): The number of samples generated per niche during
135 initialization. Defaults to 20.
136 workers (Optional[int]): Number of CPU workers/threads to parallelize the process.
137 Defaults to the number of CPUs available.
138 iterations (Optional[int]): The total number of iterations to evolve the archive.
139 Defaults to 100.
140 archive (Optional[Archive]): Pre-existing archive object for storing solutions. If `None`,
141 a new archive is initialized. Defaults to None.
142 me_params (Optional[Dict]): Additional hyperparameters specific to the map-elites algorithm.
143 Defaults to an empty dictionary.
144 cma_params (Optional[Dict]): Hyperparameters specific to the CMA-ES algorithm used
145 for evolutionary strategies. Defaults to an empty dictionary.
146 use_stats (Optional[bool]): Flag to determine whether statistical data should be
147 collected and maintained in the archive. Defaults to False.
148
149 Returns:
150 Archive: The final archive containing diverse high-performing solutions distributed
151 across the quality-diversity space.
152 """
153
154 dim = len(bounds.lb)
155 if archive is None:
156 archive = Archive(dim, qd_bounds, niche_num, use_stats)
157 archive.init_niches(samples_per_niche)
158 # initialize archive with random values
159 self.xs.view()[:] = rng.uniform(bounds.lb, bounds.ub, (niche_num, dim))
160 t0 = perf_counter()
161 qd_fitness.archive = archive # attach archive for logging
162 for iter in range(iterations):
163 archive.argsort() # sort archive to select the best_n
164 optimize_map_elites_(archive, qd_fitness, bounds, workers,
165 me_params, cma_params)
166 if is_debug_active():
167 ys = np.sort(archive.get_ys())[:100] # best 100 fitness values
168 logger.debug(f'best 100 iter {iter} best {min(ys):.3f} worst {max(ys):.3f} ' +
169 f'mean {np.mean(ys):.3f} stdev {np.std(ys):.3f} time {dtime(t0)} s')
170 return archive

267def optimize_map_elites_(archive, fitness, bounds, workers,
268 me_params, cma_params):
269 """
270 Optimize a solution using Map-Elites algorithm via multi-processing.
271
272 This function orchestrates the execution of the Map-Elites optimization
273 algorithm over multiple processes. Each process operates independently with its
274 own random generator to explore the solution space. It uses CMA-ES to update
275 solutions iteratively. The provided parameters configure the optimization
276 behavior, such as the solution archive, fitness evaluation function, search
277 bounds, and process settings specific to Map-Elites and CMA-ES.
278
279 Args:
280 archive: A data structure to store the solutions found during the
281 optimization process and their corresponding evaluation.
282 fitness: A callable function that evaluates the quality of a solution.
283 bounds: A sequence of tuples defining the lower and upper bounds for
284 each decision variable.
285 workers: An integer specifying the number of parallel processes
286 to run during optimization.
287 me_params: A dictionary or configuration object with parameters
288 for the Map-Elites algorithm.
289 cma_params: A dictionary or configuration object with parameters
290 for the Covariance Matrix Adaptation Evolution Strategy (CMA-ES).
291 """
292 sg = SeedSequence()
293 rgs = [Generator(PCG64DXSM(s)) for s in sg.spawn(workers)]
294 proc=[Process(target=run_map_elites_,
295 args=(archive, fitness, bounds, rgs[p],
296 me_params, cma_params)) for p in range(workers)]
297 [p.start() for p in proc]
298 [p.join() for p in proc]

300def run_map_elites_(archive, fitness, bounds, rg,
301 me_params, cma_params):
302 """
303 Runs the MAP-Elites algorithm with optional SBX (Simulated Binary Crossover) or
304 iso+line variation, followed by CMA-ES optimization. The algorithm iteratively
305 updates the archive with new solutions, balancing exploration and exploitation
306 over the specified number of generations.
307
308 Args:
309 archive: An archive object for storing solutions, descriptors, and their
310 fitness values, supporting niche identification and updating mechanisms.
311 fitness: Callable that computes the fitness and descriptor of a given solution.
312 bounds: Bounds object holding the lower and upper bounds for the solutions
313 during variation and optimization.
314 rg: A random generator instance used for stochastic operations within the
315 algorithm.
316 me_params: Dictionary containing tunable MAP-Elites parameters with keys
317 such as 'generations', 'chunk_size', 'use_sbx', 'dis_c', 'dis_m',
318 'iso_sigma', and 'line_sigma'.
319 cma_params: Dictionary containing CMA-ES-specific parameters such as
320 'cma_generations'.
321
322 """
323 generations = me_params.get('generations', 10)
324 chunk_size = me_params.get('chunk_size', 20)
325 use_sbx = me_params.get('use_sbx', True)
326 dis_c = me_params.get('dis_c', 20)
327 dis_m = me_params.get('dis_m', 20)
328 iso_sigma = me_params.get('iso_sigma', 0.02)
329 line_sigma = me_params.get('line_sigma', 0.2)
330 cma_generations = cma_params.get('cma_generations', 20)
331 select_n = archive.capacity
332 with threadpoolctl.threadpool_limits(limits=1, user_api="blas"):
333 for _ in range(generations):
334 if use_sbx:
335 pop = archive.random_xs(select_n, chunk_size, rg)
336 xs = variation_(pop, bounds.lb, bounds.ub, rg, dis_c, dis_m)
337 else:
338 x1 = archive.random_xs(select_n, chunk_size, rg)
339 x2 = archive.random_xs(select_n, chunk_size, rg)
340 xs = iso_dd_(x1, x2, bounds.lb, bounds.ub, rg, iso_sigma, line_sigma)
341 yds = [fitness(x) for x in xs]
342 descs = np.array([yd[1] for yd in yds])
343 niches = archive.index_of_niches(descs)
344 for i in range(len(yds)):
345 archive.set(niches[i], yds[i], xs[i])
346 archive.argsort()
347 select_n = archive.get_occupied()
348
349 for _ in range(cma_generations):
350 optimize_cma_(archive, fitness, bounds, rg, cma_params)

352def optimize_cma_(archive, fitness, bounds, rg, cma_params):
353 """
354 Optimizes a solution using the Covariance Matrix Adaptation Evolution Strategy (CMA-ES).
355 The function utilizes the process of evolutionary optimization based on objective fitness,
356 adaptive step size, and population-based approach. It leverages a provided archive to
357 introduce variations in the space of potential solutions.
358
359 Args:
360 archive: An instance of a solution archive storing previously evaluated solutions and
361 assisting in generating new candidate solutions.
362 fitness: A callable fitness function to evaluate the quality of solutions.
363 bounds: The bounds or constraints of the solution space to limit candidate solutions.
364 rg: A random generator instance used to introduce stochasticity in optimization steps.
365 cma_params: A dictionary containing configuration parameters for the CMA-ES optimizer,
366 such as population size, sigma (step size), maximum iterations, and other hyperparameters.
367
368 Raises:
369 ValueError: If parameters in cma_params are invalid (e.g., negative population size).
370 """
371 select_n = cma_params.get('best_n', 100)
372 x0, y, iter = archive.random_xs_one(select_n, rg)
373 sigma = cma_params.get('sigma',rg.uniform(0.03, 0.3)**2)
374 popsize = cma_params.get('popsize', 31)
375 es = cmaescpp.ACMA_C(archive.dim, bounds, x0 = x0,
376 popsize = popsize, input_sigma = sigma, rg = rg)
377 maxiters = cma_params.get('maxiters', 100)
378 stall_criterion = cma_params.get('stall_criterion', 5)
379 old_ys = None
380 last_improve = 0
381 for iter in range(maxiters):
382 xs = es.ask()
383 improvement, ys = update_archive(archive, xs, fitness)
384 if iter > 0:
385 if (np.sort(ys) < old_ys).any():
386 last_improve = iter
387 if last_improve + stall_criterion < iter:
388 # no improvement
389 break
390 if es.tell(improvement) != 0:
391 break
392 old_ys = np.sort(ys)

394@np.errstate(invalid='ignore')
395def update_archive(archive: Archive, xs: np.ndarray,
396 fitness: Optional[Callable[[ArrayLike], Tuple[float, np.ndarray]]] = None,
397 yds: Optional[ArrayLike] = None):
398 """
399 Evaluates a population, updates the archive with new solutions, and determines rankings.
400
401 This function processes a set of solutions and their associated descriptions or fitness
402 evaluations. The solutions are compared against an archive to determine and store better
403 solutions. It prioritizes updates to empty niches in the archive and normalizes improvements
404 for comparisons. The function finally returns the improvement values and the real fitness
405 values for the solutions.
406
407 Args:
408 archive (Archive): The archive object where solutions and corresponding fitness data are stored.
409 xs (np.ndarray): A set of solutions to be evaluated and compared against the archive.
410 fitness (Optional[Callable[[ArrayLike], Tuple[float, np.ndarray]]]):
411 A callable function that evaluates a solution and returns its fitness value and description.
412 yds (Optional[ArrayLike]): Precomputed fitness values and descriptions, if available.
413
414 Returns:
415 Tuple[np.ndarray, np.ndarray]: A tuple containing:
416 - Improvement values compared to archive elite solutions.
417 - The real fitness values evaluated for the given solutions.
418
419 Raises:
420 ValueError: If `ys` or `yds` contain invalid or mismatched content.
421 """
422 # evaluate population, update archive and determine ranking
423 popsize = len(xs)
424 if yds is None:
425 yds = [fitness(x) for x in xs]
426 descs = np.array([yd[1] for yd in yds])
427 niches = archive.index_of_niches(descs)
428 # real values
429 ys = np.fromiter((yd[0] for yd in yds), dtype=float)
430 oldys = np.fromiter((archive.get_y(niches[i]) for i in range(popsize)), dtype=float)
431 improvement = ys - oldys
432 neg = np.argwhere(improvement < 0)
433 if len(neg) > 0:
434 neg = neg.reshape((len(neg)))
435 # update archive for all real improvements
436 for i in neg:
437 archive.set(niches[i], yds[i], xs[i])
438 # prioritize empty niches
439 empty = (improvement == -np.inf) # these need to be sorted according to fitness
440 occupied = np.logical_not(empty)
441 min_valid = np.amin(improvement[occupied]) if sum(occupied) > 0 else 0
442 norm_ys = ys[empty] - np.amax(ys) - 1E-9
443 improvement[empty] = min_valid + norm_ys
444 # return both improvement compared to archive elites and real fitness
445 return improvement, ys

447@njit()
448def get_grid_indices(ds, capacity, lb, ub):
449 """
450 Computes and returns grid indices for a dataset based on the provided grid
451 capacity, lower, and upper bounds. Each point in the dataset is normalized
452 and then converted into a grid index using the specified parameters.
453
454 Args:
455 ds (np.ndarray): Input dataset (2D numpy array) where each row is a point
456 in a multi-dimensional space.
457 capacity (int): Total number of grid cells (capacity of the grid).
458 lb (np.ndarray): Lower bounds of the dataset in each dimension.
459 ub (np.ndarray): Upper bounds of the dataset in each dimension.
460
461 Returns:
462 np.ndarray: An array of grid indices, where each index corresponds to a
463 point in the input dataset.
464
465 """
466 rdim = int(capacity ** (1/ds.shape[1]) + 0.5)
467 ds_norm = (ds - lb) / (ub - lb)
468 indices = np.empty(len(ds), dtype=np.int32)
469 for i, d in enumerate(ds_norm):
470 index = 0
471 f = 1
472 for di in d:
473 index += f * int(rdim*di)
474 f *= rdim
475 indices[i] = max(0, min(capacity-1, int(index)))
476 return indices

992 def get_qd_score(self) -> float:
993 """
994 Calculates the QD (Quality-Diversity) score based on the provided array of values.
995
996 The method processes an array of values (`ys`) to compute a QD score using either the sum of reciprocal
997 values for positive values or the sum of negated negative values. The computed score is dependent on
998 whether all the values in the array are positive or some are negative.
999
1000 Returns:
1001 float: The calculated QD score.
1002 """
1003 ys = self.get_ys()
1004 occupied = (ys != np.inf)
1005 ys = ys[occupied]
1006 if len(ys) == 0:
1007 return 0
1008 min_y = np.amin(ys)
1009 if min_y > 0: # if all y > 0 use sum of reciprocal
1010 return np.sum(np.reciprocal(ys, where = ys!=0))
1011 else: # else use only the negative ones
1012 neg = (ys < 0)
1013 ys = ys[neg]
1014 return np.sum(-ys)

1406def variation_(pop, lower, upper, rg, dis_c = 20, dis_m = 20):
1407 """
1408 Generates offspring using a variation operator for a given population according to the simulated
1409 binary crossover (SBX) and polynomial mutation mechanisms. This method scales both the spread factors
1410 of crossover (`dis_c`) and mutation (`dis_m`) randomly within their range to introduce stochasticity.
1411
1412 Args:
1413 pop (np.ndarray): 2D array representing the population, where each row corresponds to an individual and
1414 each column to a decision variable.
1415 lower (np.ndarray): 1D array specifying the lower bound of each decision variable. Its length must match
1416 the number of columns in the population.
1417 upper (np.ndarray): 1D array specifying the upper bound of each decision variable. Its length must match
1418 the number of columns in the population.
1419 rg (np.random.Generator): Random generator instance for stochastic operations.
1420 dis_c (float, optional): Distribution index for crossover operation, controlling spread of solutions
1421 during SBX. Defaults to 20.
1422 dis_m (float, optional): Distribution index for mutation operation, controlling spread of mutations.
1423 Defaults to 20.
1424
1425 Returns:
1426 np.ndarray: The newly generated offspring population, with the same shape as the input `pop`.
1427 """
1428 dis_c *= 0.5 + 0.5*rg.random() # vary spread factors randomly
1429 dis_m *= 0.5 + 0.5*rg.random()
1430 pop = pop[:(len(pop) // 2) * 2][:]
1431 (n, d) = np.shape(pop)
1432 parent_1 = pop[:n // 2, :]
1433 parent_2 = pop[n // 2:, :]
1434 beta = np.zeros((n // 2, d))
1435 mu = rg.random((n // 2, d))
1436 beta[mu <= 0.5] = np.power(2 * mu[mu <= 0.5], 1 / (dis_c + 1))
1437 beta[mu > 0.5] = np.power(2 * mu[mu > 0.5], -1 / (dis_c + 1))
1438 beta = beta * ((-1)** rg.integers(2, size=(n // 2, d)))
1439 beta[rg.random((n // 2, d)) < 0.5] = 1
1440 parent_mean = (parent_1 + parent_2) / 2
1441 parent_diff = (parent_1 - parent_2) / 2
1442 offspring = np.vstack((parent_mean + beta * parent_diff, parent_mean - beta * parent_diff))
1443 site = rg.random((n, d)) < 1.0 / d
1444 mu = rg.random((n, d))
1445 temp = site & (mu <= 0.5)
1446 lower, upper = np.tile(lower, (n, 1)), np.tile(upper, (n, 1))
1447 norm = (offspring[temp] - lower[temp]) / (upper[temp] - lower[temp])
1448 offspring[temp] += (upper[temp] - lower[temp]) * \
1449 (np.power(2. * mu[temp] + (1. - 2. * mu[temp]) * np.power(np.abs(1. - norm), dis_m + 1.),
1450 1. / (dis_m + 1)) - 1.)
1451 temp = site & (mu > 0.5)
1452 norm = (upper[temp] - offspring[temp]) / (upper[temp] - lower[temp])
1453 offspring[temp] += (upper[temp] - lower[temp]) * \
1454 (1. - np.power(
1455 2. * (1. - mu[temp]) + 2. * (mu[temp] - 0.5) * np.power(np.abs(1. - norm), dis_m + 1.),
1456 1. / (dis_m + 1.)))
1457 return np.clip(offspring, lower, upper)

1459def iso_dd_(x1, x2, lower, upper, rg, iso_sigma = 0.01, line_sigma = 0.2):
1460 """
1461 Generates isotropic deviation data within specified bounds.
1462
1463 This function creates a variation of the input data `x1` influenced by
1464 Gaussian noise parameters for isotropic and line deviations. The result
1465 is clipped to remain within the range defined by the `lower` and `upper`
1466 bounds.
1467
1468 Args:
1469 x1: The primary input array for generating isotropic deviation data.
1470 x2: The secondary input array influencing line deviation calculation.
1471 lower: The lower bound for clipping the resulting data.
1472 upper: The upper bound for clipping the resulting data.
1473 rg: Random generator object used for generating Gaussian noise.
1474 iso_sigma: Standard deviation of the isotropic Gaussian noise. Defaults
1475 to 0.01.
1476 line_sigma: Standard deviation of the line Gaussian noise. Defaults to
1477 0.2.
1478
1479 Returns:
1480 ndarray: The processed array with isotropic and line deviations
1481 applied and bounded by the given limits.
1482
1483 """
1484 a = rg.normal(0, iso_sigma, x1.shape)
1485 b = rg.normal(0, line_sigma, x2.shape)
1486 z = x1 + a + np.multiply(b, (x1 - x2))
1487 return np.clip(z, lower, upper)
