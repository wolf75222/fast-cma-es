15 Description:
16 - Eigen based implementation of multi objective
17 Differential Evolution using the DE/pareto/1 strategy.
18 Derived and adapted for MO from its C++ counterpart
19 [2]
20
21 Can switch to NSGA-II like population update via parameter 'nsga_update'.
22 Then it works essentially like NSGA-II but instead of the tournament selection
23 the whole population is sorted and the best individuals survive. To do this
24 efficiently the crowd distance ordering is slightly inaccurate.
25
26 Supports parallel fitness function evaluation.
27
28 Features enhanced multiple constraint ranking [3]
29 improving its performance in handling constraints for engineering design optimization.
30
31 Enables the comparison of DE and NSGA-II population update mechanism with everything else
32 kept completely identical.
33
34 Requires python 3.5 or higher.
35
36 Uses the following deviation from the standard DE algorithm:
37 a) oscillating CR/F parameters.
38
39 You may keep parameters F and CR at their defaults since this implementation works well with the given settings for most problems,
40 since the algorithm oscillates between different F and CR settings.
41
42 For expensive objective functions (e.g. machine learning parameter optimization) use the workers
43 parameter to parallelize objective function evaluation. This causes delayed population update.
44 It is usually preferrable if popsize > workers and workers = mp.cpu_count() to improve CPU utilization.
45
46 The ints parameter is a boolean array indicating which parameters are discrete integer values. This
47 parameter was introduced after observing non optimal DE-results for the ESP2 benchmark problem:
48 [4]
49 If defined it causes a "special treatment" for discrete variables: They are rounded to the next integer value and
50 there is an additional mutation to avoid getting stuck to local minima.
51
52 See [5] for a detailed description.

182def retry(mofun: Callable[[ArrayLike], ArrayLike],
183 nobj: int,
184 ncon: int,
185 bounds: Bounds,
186 guess: Optional[np.ndarray] = None,
187 num_retries: Optional[int] = 64,
188 popsize: Optional[int] = 64,
189 max_evaluations: Optional[int] = 100000,
190 workers: Optional[int] = mp.cpu_count(),
191 nsga_update: Optional[bool] = False,
192 pareto_update: Optional[int] = 0,
193 ints: Optional[ArrayLike] = None,
194 capacity: Optional[int] = None):
195
196 """
197 Retries a multi-objective optimization process in parallel to optimize given
198 objective functions and constraints.
199
200 This function orchestrates the process of executing an optimization task
201 multiple times across multiple workers, each working with different random
202 seeds. It uses a population-based approach to iteratively search for optimal
203 solutions for a given multi-objective problem, leveraging parallel computing
204 to efficiently handle large workloads.
205
206 Args:
207 mofun (Callable[[ArrayLike], ArrayLike]): The objective function to optimize,
208 mapping input parameters to objective and constraint values.
209 nobj (int): Number of objective functions in the optimization problem.
210 ncon (int): Number of constraints in the optimization problem.
211 bounds (Bounds): The bounds for the decision variables. This defines the
212 lower and upper bounds for optimization.
213 guess (Optional[np.ndarray]): Initial guess for the input variables.
214 Default is None.
215 num_retries (Optional[int]): Number of retries allowed for each worker.
216 Default is 64.
217 popsize (Optional[int]): The population size for the optimization algorithm.
218 Default is 64.
219 max_evaluations (Optional[int]): Maximum number of function evaluations
220 allowed. Default is 100000.
221 workers (Optional[int]): Number of workers to run in parallel. If not
222 specified, it defaults to the number of CPUs available on the machine.
223 nsga_update (Optional[bool]): If True, enables an additional NSGA update
224 step in the optimization. Default is False.
225 pareto_update (Optional[int]): Frequency of updating the Pareto front
226 during optimization. Default is 0 (no updates).
227 ints (Optional[ArrayLike]): Indices of decision variables that are integers.
228 Default is None.
229 capacity (Optional[int]): Capacity of the storage system for maintaining
230 results during optimization. If not provided, it defaults to
231 2048 times the population size.
232
233 Returns:
234 Tuple[np.ndarray, np.ndarray]: A tuple containing:
235 - xs: The input configurations corresponding to the Pareto-optimal solutions.
236 - ys: The objective and constraint evaluation results for the
237 Pareto-optimal solutions.
238 """
239
240 dim, _, _ = _check_bounds(bounds, None)
241 if capacity is None:
242 capacity = 2048*popsize
243 store = mode.store(dim, nobj + ncon, capacity)
244 sg = SeedSequence()
245 rgs = [Generator(PCG64DXSM(s)) for s in sg.spawn(workers)]
246 proc=[Process(target=_retry_loop,
247 args=(num_retries, pid, rgs, mofun, nobj, ncon, bounds, guess, popsize,
248 max_evaluations, workers, nsga_update, pareto_update,
249 store, ints))
250 for pid in range(workers)]
251 [p.start() for p in proc]
252 [p.join() for p in proc]
253 xs, ys = store.get_front()
254 return xs, ys

256def _retry_loop(num_retries, pid, rgs, mofun, nobj, ncon, bounds, guess, popsize,
257 max_evaluations, workers, nsga_update, pareto_update,
258 store, ints):
259 """
260 Executes a retry loop for parallel optimization tasks, ensuring multiple
261 minimization attempts are conducted until a sufficient number of results are
262 added to the storage.
263
264 Args:
265 num_retries: Number of retry attempts for the optimization loop.
266 pid: Process identifier used for dealing with random generators.
267 rgs: List of random number generators for each process.
268 mofun: Multi-objective function to be minimized.
269 nobj: Number of objectives in the optimization problem.
270 ncon: Number of constraints in the optimization problem.
271 bounds: Boundaries for the decision variables in the optimization problem.
272 guess: Initial guess values for the optimization variables.
273 popsize: Population size for the optimization algorithm.
274 max_evaluations: Maximum number of evaluations for each optimization attempt.
275 workers: Number of worker processes available for parallelization.
276 nsga_update: Callback or function for handling NSGA updates during optimization.
277 pareto_update: Callback or function for managing Pareto updates.
278 store: Storage object for managing results and tracking progress.
279 ints: Indices of decision variables that are integer-constrained.
280 """
281 store.create_views()
282 t0 = time.perf_counter()
283 num = max(1, num_retries - workers)
284 with threadpoolctl.threadpool_limits(limits=1, user_api="blas"):
285 while store.num_added.value < num:
286 minimize(mofun, nobj, ncon, bounds, guess, popsize,
287 max_evaluations = max_evaluations,
288 nsga_update=nsga_update, pareto_update=pareto_update,
289 rg = rgs[pid], store = store, ints=ints)
290 if is_debug_active():
291 logger.debug("retries = {0}: time = {1:.1f} i = {2}"
292 .format(store.num_added.value, dtime(t0), store.num_stored.value))

296 A class for managing the MODE-C optimization algorithm.
297
298 This class provides an interface for the multi-objective differential evolution
299 (MODE) algorithm and supports handling of objective functions, constraints, parallel
300 evaluation, and population management. It is designed to work with scenarios that
301 require solving optimization problems with multiple competing objectives, bounded
302 variables, and potentially integer-constrained decision variables.
303
304 Attributes:
305 popsize (int): The size of the population used in the optimization process.
306 dim (int): The dimensionality of the decision variable space.
307 nobj (int): The number of objective functions.
308 ncon (int): The number of constraints in the optimization problem.
309 bounds (Bounds): The bounds on variables, specified as a sequence of (min, max) bounds or

312 def __init__(self,
313 nobj: int,
314 ncon: int,
315 bounds: Bounds,
316 popsize: Optional[int] = 64,
317 f: Optional[float] = 0.5,
318 cr: Optional[float] = 0.9,
319 pro_c: Optional[float] = 0.5,

330
331 """
332 Initializes an instance of the optimization problem using the specified parameters.
333
334 This constructor sets up the necessary parameters for a multi-objective differential
335 evolution (MODE) algorithm, with options for NSGA-II-inspired updates and Pareto
336 front approximations. It handles checks on bounds, dimensions, and population size,
337 and initializes the required internal structures for the optimization process.
338
339 Args:
340 nobj (int): Number of objectives in the optimization problem.
341 ncon (int): Number of constraints in the optimization problem.
342 bounds (Bounds): Object defining the lower and upper bounds for the decision variables.
343 popsize (Optional[int]): Size of the population. Defaults to 64.
344 f (Optional[float]): Differential weight utilized in mutation. Defaults to 0.5.
345 cr (Optional[float]): Crossover probability. Defaults to 0.9.
346 pro_c (Optional[float]): Probability of crossover used in simulated binary crossover. Defaults to 0.5.
347 dis_c (Optional[float]): Distribution index for crossover. Defaults to 15.0.
348 pro_m (Optional[float]): Probability of mutation. Defaults to 0.9.
349 dis_m (Optional[float]): Distribution index for mutation. Defaults to 20.0.
350 nsga_update (Optional[bool]): Flag to enable NSGA-II-inspired update rules. Defaults to True.
351 pareto_update (Optional[int]): Mode for Pareto front update. Defaults to 0.
352 ints (Optional[ArrayLike]): Binary array denoting whether each variable is an integer (True) or continuous (False).
353 Defaults to None.
354 min_mutate (Optional[float]): Minimum mutation rate for adaptive mutation. Defaults to 0.1.
355 max_mutate (Optional[float]): Maximum mutation rate for adaptive mutation. Defaults to 0.5.
356 rg (Optional[Generator]): Random number generator instance. Defaults to Generator(PCG64DXSM()).
357 runid (Optional[int]): Unique identifier for the model run. Defaults to 0.
358
359 """
360
361 dim, lower, upper = _check_bounds(bounds, None)
362 if popsize is None:
363 popsize = 64
364 if popsize % 2 == 1 and nsga_update: # nsga update requires even popsize
365 popsize += 1
366 if lower is None:
367 lower = [0]*dim
368 upper = [0]*dim
369 if ints is None or nsga_update: # nsga update doesn't support mixed integer
370 ints = [False]*dim
371 array_type = ct.c_double * dim
372 bool_array_type = ct.c_bool * dim
373 seed = int(rg.uniform(0, 2**32 - 1))
374 try:
375 self.ptr = initMODE_C(runid, dim, nobj, ncon, seed,
376 array_type(*lower), array_type(*upper), bool_array_type(*ints),
377 popsize, f, cr,
378 pro_c, dis_c, pro_m, dis_m,
379 nsga_update, pareto_update, min_mutate, max_mutate)
380 self.popsize = popsize
381 self.dim = dim
382 self.nobj = nobj
383 self.ncon = ncon
384 self.bounds = bounds
385 except Exception as ex:
386 print (str(ex))
387 pass

403 """
404 Set the initial guess values for optimization along with

408 on the input guess and the provided function evaluator.

560 def minimize_ser(self,
561 fun: Callable[[ArrayLike], ArrayLike],
562 max_evaluations: Optional[int] = 100000) -> Tuple[np.ndarray, np.ndarray]:
563 """
564 Minimizes the given scalar function using a serial optimization method. The function
565 iteratively evaluates the objective function on the candidate solutions, updates the
566 internal state of the optimizer, and stops when the termination criterion is met or the
567 maximum number of evaluations is reached.
568
569 Args:
570 fun (Callable[[ArrayLike], ArrayLike]): The objective function to be minimized.
571 It should take an input of type ArrayLike and return a value of type ArrayLike.
572 max_evaluations (Optional[int]): The maximum number of function evaluations allowed.
573 Defaults to 100000.
574
575 Returns:
576 Tuple[np.ndarray, np.ndarray]: A tuple containing the final candidate solutions as a
577 numpy array and their corresponding evaluated objective function values as a numpy array.
578 """
579 evals = 0
580 stop = 0
581 while stop == 0 and evals < max_evaluations:
582 xs = self.ask()
583 ys = np.array([fun(x) for x in xs])
584 stop = self.tell(ys)
585 evals += self.popsize
586 return xs, ys

588 def minimize_par(self,
589 fun: Callable[[ArrayLike], ArrayLike],
590 max_evaluations: Optional[int] = 100000,
591 workers: Optional[int] = mp.cpu_count()) -> Tuple[np.ndarray, np.ndarray]:
592 """
593 Run a parallel multi-objective optimization process.
594
595 This method leverages parallel processing to perform multi-objective
596 optimizations using a given objective function. It runs multiple iterations
597 until the stopping criteria are met or the specified maximum evaluations are
598 reached.
599
600 Args:
601 fun (Callable[[ArrayLike], ArrayLike]): Objective function to minimize.
602 max_evaluations (Optional[int]): Maximum number of evaluations allowed.
603 Defaults to 100000.
604 workers (Optional[int]): Number of parallel workers to use. Defaults to
605 the number of CPU cores.
606
607 Returns:
608 Tuple[np.ndarray, np.ndarray]: A tuple containing the array of evaluated
609 input solutions (`xs`) and their corresponding objective values (`ys`).
610
611 """
612 fit = parallel_mo(fun, self.nobj + self.ncon, workers)
613 evals = 0
614 stop = 0
615 while stop == 0 and evals < max_evaluations:
616 xs = self.ask()
617 ys = fit(xs)
618 stop = self.tell(ys)
619 evals += self.popsize
620 fit.stop()
621 return xs, ys
