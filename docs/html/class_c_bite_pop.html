Detailed Description

template<typename ptype>
class CBitePop< ptype >

Class implements storage of population parameter vectors, costs, centroid, and ordering. Note that the derived classes should delete allocated buffers in their destructors, not relying on a single deleteBuffers() call.

Template Parameters
ptypeParameter value storage type.

◆ initBuffers()

template<typename ptype>
virtual void CBitePop< ptype >::initBuffers ( const int aParamCount,
const int aPopSize,
const int aCnsCount = 0,
const int aObjCount = 1 )
inlinevirtual

Function initializes all common buffers, and "PopSize" variables. This function should be called when population's dimensions were changed. This function calls the deleteBuffers() function to release any derived classes' allocated buffers. Allocates an additional vector for temporary use, which is at the same the last vector in the PopParams array. Derived classes should call this function of the base class.

Parameters
aParamCountNew parameter count.
aPopSizeNew population size. If <= 0, population buffers will not be allocated.
aCnsCountNew constraint value count.
aObjCountNew objective value count. If equals 0, a rank element will not be auto-added.

Reimplemented in CBiteOptBase< ptype >, CBiteOptBase< double >, CBiteOptBase< int64_t >, CBiteOrt, CNMSeqOpt, and CSpherOpt.

◆ updatePop()

template<typename ptype>
int CBitePop< ptype >::updatePop ( double UpdCost,
const ptype *const UpdParams,
const bool DoUpdateCentroid,
const bool CanRejectCost = true )
inline

Function replaces the highest-cost previous solution, updates centroid. This function considers the value of the CurPopPos variable - if it is smaller than the PopSize, the new solution will be added to population without any checks.

Parameters
UpdCostCost of the new solution.
UpdParamsNew parameter values.
DoUpdateCentroid"True" if centroid should be updated using running sum. This update is done for parallel populations.
CanRejectCostIf "true", solution with a duplicate cost will be rejected; this may provide a performance improvement. Solutions can only be rejected when the whole population was filled.
Returns
Insertion position, ">=CurPopSize" if the cost constraint was not met.
