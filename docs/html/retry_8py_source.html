66def minimize(fun: Callable[[ArrayLike], float],
67 bounds: Bounds,
68 value_limit: Optional[float] = np.inf,
69 num_retries: Optional[int] = 1024,
70 workers: Optional[int] = mp.cpu_count(),
71 popsize: Optional[int] = 31,
72 max_evaluations: Optional[int] = 50000,
73 capacity: Optional[int] = 500,
74 stop_fitness: Optional[float] = -np.inf,
75 optimizer: Optional[Optimizer] = None,
76 statistic_num: Optional[int] = 0,
77 ) -> OptimizeResult:
78 """
79 Minimizes a given objective function using the specified optimizer and stores the progress.
80
81 This function attempts to find the minimum value of a given function within the bounds
82 provided. It incorporates retry logic to handle multiple optimization attempts in case
83 of failure or suboptimal results. The optimization process can also operate in a multiprocess
84 environment with a configurable number of workers. Additionally, progress data is stored
85 to analyze statistical results throughout the optimization.
86
87 Args:
88 fun: The objective function to minimize. The function must accept an array-like input
89 and return a scalar float as the function value.
90 bounds: Bounds for the input variables of the objective function. Defines the
91 acceptable search space during optimization.
92 value_limit: Upper limit for the acceptable function value. Defaults to infinity.
93 After reaching this limit, retries are triggered if configured.
94 num_retries: Number of times the optimization process should be retried upon failure
95 or suboptimal results. Defaults to 1024.
96 workers: The number of parallel workers to use for optimization. Defaults to the
97 number of CPU cores available.
98 popsize: Population size for the optimization process. Determines the number
99 of potential solutions considered at each iteration.
100 max_evaluations: The maximum number of evaluations allowed in the optimization
101 process. This is a hard limit on computational effort.
102 capacity: The storage capacity for retaining past evaluation results or progress
103 data. Used for analyzing statistics or performance.
104 stop_fitness: The target fitness value for stopping the optimization. When a result
105 meets or exceeds this fitness value, the process terminates. Defaults to
106 negative infinity.
107 optimizer: An optimization algorithm instance to drive the optimization process.
108 If not provided, a default optimizer is created based on `de_cma`.
109 statistic_num: The statistical metric number to associate with the optimization
110 data. Used for tracking or analyzing statistical trends.
111
112 Returns:
113 OptimizeResult: The result of the optimization process, including details about
114 the optimal solution, number of function evaluations, and success status.
115 """
116
117 if optimizer is None:
118 optimizer = de_cma(max_evaluations, popsize, stop_fitness)
119 store = Store(fun, bounds, capacity = capacity, statistic_num = statistic_num)
120 return retry(store, optimizer.minimize, num_retries, value_limit, workers, stop_fitness)

122def retry(store: Store,
123 optimize: Callable,
124 num_retries: int,
125 value_limit: Optional[float] = np.inf,
126 workers: Optional[int] = mp.cpu_count(),
127 stop_fitness: Optional[float] = -np.inf) -> OptimizeResult:
128 """
129 Retries optimization multiple times using the specified number of worker processes. Each worker
130 conducts optimization in parallel and contributes to the shared results store. After all workers
131 complete their tasks, the stored results are sorted, persisted, and the best result is returned.
132
133 Args:
134 store (Store): A shared result store object that collects optimization results from all workers.
135 optimize (Callable): A callable function that performs the optimization for a single run.
136 num_retries (int): The number of retries/iterations each worker should perform.
137 value_limit (Optional[float]): The upper limit for objective values to consider valid. Defaults
138 to positive infinity.
139 workers (Optional[int]): The number of worker processes to spawn for parallel optimization.
140 Defaults to the number of CPU cores available.
141 stop_fitness (Optional[float]): The fitness value at which the optimization can early stop.
142 Defaults to negative infinity.
143
144 Returns:
145 OptimizeResult: An object containing the best solution found, its objective value, number of
146 function evaluations, and a success status.
147 """
148 sg = SeedSequence()
149 rgs = [Generator(PCG64DXSM(s)) for s in sg.spawn(workers)]
150 proc=[Process(target=_retry_loop,
151 args=(pid, rgs, store, optimize, num_retries, value_limit, stop_fitness)) for pid in range(workers)]
152 [p.start() for p in proc]
153 [p.join() for p in proc]
154 store.sort()
155 store.dump()
156 return OptimizeResult(x=store.get_x_best(), fun=store.get_y_best(),
157 nfev=store.get_count_evals(), success=True)

382 """
383 Manages and tracks the optimization process by storing evaluation results, maintaining statistics,

385
386 This class is designed for optimization tasks, allowing the storage of evaluated points,
387 their objective function values, and statistical tracking of improvements. It supports
388 multi-processed environments using shared memory for data storage.

390 Attributes:

392 lower (ArrayLike): The lower bounds of the objective function arguments.
393 upper (ArrayLike): The upper bounds of the objective function arguments.
394 capacity (int): Maximum capacity of the evaluation store.
395 check_interval (int): Number of iterations after which the evaluation memory is sorted.
396 dim (int): Dimensionality of the problem space.
397 xs (Shared2d): Shared 2D array for storing the evaluated points in the search space.
398 ys (multiprocessing.RawArray): Shared array for storing the corresponding function values.

497 def get_improvements(self):
498 """
499 Generates an array of improvements based on time and value up to a specific index.
500
501 This method combines the time and value attributes into a NumPy array of tuples,
502 containing the data up to the si attribute's value. The method assumes that the
503 attributes `time` and `val` are iterable and contain corresponding elements. The

519 # get num best values at evenly distributed times
520 def get_statistics(self, num: int) -> List:
521 """
522 Calculates statistics for the provided number of samples by segmenting
523 and mapping values based on time intervals.
524
525 This method segments a range of time into equal intervals based on
526 the given number of samples. It then maps each interval to the
527 corresponding value at the closest timestamp within that interval.
528
529 Args:
530 num (int): The number of intervals or samples for which statistics
531 are calculated.
532
533 Returns:
534 List: A list of values corresponding to the calculated statistics
535 for each interval.
536 """
537 ts = self.time[:self.si.value]
538 ys = self.val[:self.si.value]
539 mt = ts[-1]
540 dt = 0.9999999 * mt / num
541 conv = []
542 ti = 0
543 val = ys[0]
544 for i in range(num):
545 while ts[ti] < (i+1) * dt:
546 ti += 1
547 val = ys[ti]
548 conv.append(val)
549 return conv

582 def sort(self) -> int: # sort all entries to make room for new ones, determine best and worst
583 """
584 Sort entries to retain the best ones, thereby optimizing storage for new entries.
585
586 This method sorts the stored entries based on their associated values, retains the top
587 90% (as determined by the capacity), and updates the internal data structures to reflect
588 the new ordering and count of retained entries. It subsequently returns the updated
589 number of stored entries.
590
591 Returns:
592 int: The number of stored entries after sorting and truncating to the top 90%.
593
594 """
595 ns = self.num_stored.value
596 ys = np.asarray(self.ys[:ns])
597 yi = ys.argsort()
598 numStored = min(ns, int(0.9*self.capacity)) # keep 90% best
599 self.xs_view[:numStored] = self.xs_view[yi][:numStored]
600 self.ys[:numStored] = ys[yi][:numStored]
601 self.num_stored.value = numStored
602 return numStored

610 Args:

630 self.sort()
631 cnt = self.count_stat_runs.value
632 diff = min(1E20, y - self.mean.value) # avoid overflow

634 self.mean.value += diff / cnt
635 ns = self.num_stored.value
636 self.num_stored.value = ns + 1
637 self.xs_view[self.num_stored.value, :] = x
638 self.ys[self.num_stored.value] = y
639 if is_debug_active():
640 dt = dtime(self.t0)
641 message = '{0} {1} {2} {3} {4:.6f} {5:.6f} {6:.2f} {7:.2f}'.format(
642 dt, int(self.count_evals.value / dt), self.count_runs.value, self.count_evals.value, \
643 y, self.best_y.value, self.get_y_mean(), self.get_y_standard_dev())

832 """
833 Executes a retry loop for optimization, managing retries and applying constraints
834 based on fitness value and evaluation limits.
835
836 Args:
837 pid (int): Process ID or thread identifier for parallel processing.
838 rgs (list): List of random generators, corresponding to each process or thread.
839 store (object): A storage or result management object handling bounds,
840 results, statistics, and fitness evaluations.
841 optimize (callable): Optimization function to be executed during the loop.
842 num_retries (int): Maximum number of retries allowed during the optimization process.
843 value_limit (float): Constraint limit for the solution value to validate results.
844 stop_fitness (float, optional): Fitness threshold that, once achieved, stops
845 further processing. Defaults to negative infinity.
846 """
847 store.create_xs_view()
848 fun = store.wrapper if store.statistic_num > 0 else store.fun
849 lower = store.lower
850 with threadpoolctl.threadpool_limits(limits=1, user_api="blas"):
851 while store.get_runs_compare_incr(num_retries) and store.best_y.value > stop_fitness:
852 try:
853 rg = rgs[pid]
854 sol, y, evals = optimize(fun, Bounds(store.lower, store.upper), None,
855 [rg.uniform(0.05, 0.1)]*len(lower), rg, store)
856 store.add_result(y, sol, evals, value_limit)
857 except Exception as ex:
858 print(str(ex))
