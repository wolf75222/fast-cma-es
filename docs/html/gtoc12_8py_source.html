1'''
2GTOC (Global Trajectory Optimization Competition) was initiated 20 years ago by ESA to
3provide a platform for Space Agencies to exchange ideas how to plan space flights.
4Every one or two years the best space trajectory planners in the world, including
5JPL (NASA) and ESA, compete solving a complex task. This example is related to GTOC12,
6which is about planning the mining of asteroids as efficiently as possible.
7
8Each ship departs from and returns to earth, visiting a number of asteroids in between.
9At the first visit a miner is dropped, at the second visit the mined material is collected.
10Each miner collects 10 kg material per year.
11Only one miner per asteroid is allowed, and there are 60000 asteroids to choose from.
12Goal is to maximize the average material mined by each ship.
13The whole mining operation is limited to 15 years (epoch 64328 - epoch 69807 MJD),
14see https://gtoc12.tsinghua.edu.cn/competition/theProblem for the original task description.
15
16This example solves a subtask of GTOC12,
17For a cluster of 44 Asteroids taken from JPLs winning solution
18we computed a full asteroid-asteroid transfer graph. This graph
19excludes transfers/edges which are too early / too late to facilitate earth->asteroid
20or asteroid->earth transfers.
21
22A possible transfer is defined by the tuple (a1, a2, dm, ep, tof) where:
23
24Graph nodes:
25a1: start asteroid of the transfer
26a2: target asteroid of the transfer
27
28Graph edge attributes:
29dm: delta propellant mass, propellant consumption
30ep: start epoch (day)
31tof: time of flight, ep + tof is the arrival epoch
32
33For a fixed number of ships, the task is now to maximize the mined material (per ship)
34by finding graph paths corresponding to the ship trajectories fulfilling all
35constraints:
36
37- A ship may not start heading for the next asteroid before it arrives at an asteroid.
38- Propellant consumption (sum of the dm values along the path) is limited (<= 1500 kg) for each ship.
39- Each asteroid can be visited at most twice.
40- If a ship arrives at an asteroid which was visited before, it collects 10*y kg material,
41 where y is the time in years since the first visit.
42- If there is no edge to any target asteroid after the arrival time or propellant
43 consumption would exceed the limit the journey for the ship ends.
44
45This task, although quite similar to ESAs Wormhole task
46https://optimise.esa.int/challenge/spoc-2-wormhole-transportation-network/p/wormhole-transportation-network
47differs in several aspects:
48
49- Much lower number of nodes - only 44 instead of 10000.
50- All ships (and not only two) are relevant.
51- Number of edges is similar - 246736, but we have for each asteroid 4000-8000 outgoing edges
52 and on average 130 (up to 444) edges between any two asteroids.
53- Any node can be used as start / target node.
54
55Independent from GTOC12 the subtask shown here represents an interesting variation of the
56Multiple Vehicle Routing Problem (https://developers.google.com/optimization/routing/vrp),
57where instead of fixed distances between targets/asteroids there are many different time
58dependent opportunities to fly from one asteroid to another. These opportunities differ
59both in time of flight and propellant consumption. Solving this class of problems can
60assist the planning of complex space flight operations involving many ships.
61Beside asteroid mining other applications could be space debris removal or maintenance/repair
62of large satellite networks.
63
64The task is at the same time quite simple and very hard to solve using search.
65Problem is the "coordination" of dropping the miner and collecting the mined material
66between ships.
67
68If we expand the branches for all ships simultaneously the branching factor explodes.
69But if the ships are processed sequentially it is very hard to come up with a good heuristic
70which branches to eliminate.
71
72Here we use continuous evolutionary optimization - the BiteOpt meta-algorithm which
73dynamically adapts the population update method.
74
75The objective fitness function looks as follows:
76
77- Let n = 44 be the number of asteroids in the cluster / the number of nodes.
78- X is the argument vector containing 2*n=88 decision variables in the interval [0,1].
79- p = numpy.argsort(X) is a permutation of the integers between 0 and 88 dependent
80 on the argument vector X.
81- seq = [i % n for i in p] is a sequence of 88 asteroid indices where each asteroid occurs exactly twice.
82- remove asteroids from seq, starting from the first one (including its 2nd occurrence)
83 until the asteroid sequence has a specific length l.
84
85We end up with an asteroid sequence of length l depending on the argument vector X which contains
86each asteroid exactly twice. l depends on the number of ships n and should be adjusted performing
87test runs of the optimization. l limits the path length for each ship, but if chosen too high
88the optimization itself has to ensure there are only asteroid pairs in all paths.
89
90Now we follow the graph using the sequence determined by the decision variables as "guidance".
91We always choose the first possible transfer between two asteroids/nodes.
92If there is no such transfer - or the ships fuel consumption would be above the limit, we
93"start" the next ship / path. If there is no more ship available we compute the mined material
94and use this as objective to maximize. Additional objectives are propellant consumption
95and the sum of the length of all paths added using the
96weighted sum approach. Although we are only interested in the mined material, the other
97objectives serve as an heuristic "guiding" the optimization.
98
99The result can further be enhanced by applying a second optimization (not include here,
100left as exercise). This time the asteroid sequence is fixed, but instead of the first
101possible transfer we choose the nth one, where n is selected according the continuous
102argument vector. If n is in the interval [1, n_max] we convert the continuous argument vector
103to integer values using ns = X.astype(int). You need to define a function
104'_transfer_n(n, a2, epoch_min, transfers, idx)' selecting the nth instead of the first
105possible transfer, and a function 'eval_sequence_n(sequence, ns)' which applies it instead of
106'_transfer(a2, epoch_min, transfers, idx)'. As optimizer parallel BiteOpt can be applied again.
107
108Our chosen propellant limit of 1500kg may be a bit high if you consider mined mass and
109the propellant needed for the transfers from and to earth. Therefore we need the
110second optimization phase to further reduce propellant consumption. The weighted sum of the
111objective function needs to be adapted accordingly.
112'''

138@njit(fastmath=True, cache=True)
139def _mined(asteroids, epochs): # mined material for a sequence of asteroids and epochs
140 mined = 0
141 for i in range(len(asteroids)-1):
142 a1 = asteroids[i]
143 for j in range(i+1, len(asteroids)):
144 if asteroids[j] == a1:
145 mined += abs(epochs[j] - epochs[i])
146 break
147 return 10*mined/365.25 # mined per year
148
149@njit(fastmath=True, cache=True) # reduce sequence to ast_used asteroids maintaining asteroid pairs
150def _reduce(ast_num, ast_used, sequence0): # use first
151 used = np.zeros(ast_num, dtype = numba.boolean)
152 sequence = np.empty(ast_used, dtype = numba.int32)
153 j = 0 # number in sequence
154 n = 0 # number of asteroids used
155 for i in range(len(sequence0)):
156 asteroid = sequence0[i]
157 if used[asteroid]:
158 sequence[j] = asteroid # always take asteroid already used
159 j += 1
160 else:
161 if n < ast_used: # only take new asteroid if not already ast_used taken
162 used[asteroid] = True # mark asteroid as used
163 n += 2
164 sequence[j] = asteroid
165 j += 1
166 if j >= len(sequence):
167 return sequence
168 return None

170@njit(fastmath=True, cache=True)
171def _transfer(a2, epoch_min, transfers, idx): # finds the next possible transfer to asteroid a2
172 for i in idx: # indices are used to speed up graph traversal
173 if transfers[i][2] == a2:
174 for transfer in transfers[i+1:]: # numba is essential for performance here
175 if transfer[0] < 0: # determines the asteroids
176 return None
177 epoch = transfer[1]
178 if epoch < epoch_min:
179 continue
180 return transfer
181 return None

197 transfers = Transfers(fname)
198 best_mined = mp.RawValue(ct.c_double, 0)
199 best_y = mp.RawValue(ct.c_double, 1E99)
200 t0 = time.perf_counter()
201 evals = mp.RawValue(ct.c_int, 0)

207 dim = 2*ast_num

209 workers = mp.cpu_count() # we use all CPU cores

211 def fit(x):
212 permutation = np.fromiter((a % ast_num for a in np.argsort(x)), dtype = int)
213 sequence = _reduce(ast_num, ast_used, permutation) # shorten the sequence of asteroids
214 return eval_sequence(sequence)

216 def eval_sequence(sequence):
217 try:
218 ships = 0
219 epoch = 0
220 mass = 0
221 all_mass = []
222 y = 0
223 epochs = []
224 asteroids = []
225 start_asts = []
226 for i in range(len(sequence)-1):
227 a1 = sequence[i]
228 a2 = sequence[i+1]
229 # transfer = delta_mass, epoch_start, time_of_flight
230 transfer = transfers.get_transfer(a1, a2, epoch)
231 if transfer is None or mass + transfer[0] > 1500: # limit delta mass to 1500 kg
232 ships += 1 # last path ended, start a new ship / path
233 epoch = 0
234 all_mass.append(mass) # accumulate propellant consumption for all paths
235 mass = 0
236 if ships >= ship_used:
237 break # no ship left
238 else:
239 delta_mass, epoch_start, time_of_flight = transfer
240 if epoch == 0: # first asteroid of a path (coming from earth)
241 epochs.append(epoch_start)
242 asteroids.append(a1)
243 start_asts.append(a1)
244 epoch = epoch_start + time_of_flight
245 asteroids.append(a2) # target asteroid a2
246 epochs.append(epoch)
247 mass += delta_mass # accumulate propellant consumption for transfers
248 if mass > 0: # increase ast_used if this happens, sequence is too short
249 all_mass.append(mass)
250 mined = _mined(np.array(asteroids), np.array(epochs)) / ship_used
251 y = -mined + 0.1*np.mean(all_mass) - 2*len(asteroids)
252 evals.value += 1
253 if y < best_y.value or mined > best_mined.value:
254 dt = max(1E-9, dtime(t0)) # avoid division by 0
255 am = ",".join(f'{int(m)}' for m in all_mass)
256 print(f't = {dt} ev = {evals.value} ev/s = {evals.value/dt:.3f} y = {y:.3f} mined = {mined:.2f} dm = {am} a = {len(asteroids)} {list(sequence)}')
257 if y < best_y.value:
258 best_y.value = y
259 if mined > best_mined.value:
260 best_mined.value = mined
261 return y
262
263 except Exception as ex:
264 return 100000

266 if seq is None: # optimize using parallel BiteOpt
267 retry.minimize_plot("minimize_plot", Bite_cpp(max_evals, M=6), fit, bounds, num_retries=num_restarts, workers=workers,
